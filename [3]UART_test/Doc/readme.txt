# STM32 UART 串口通信实验项目

## 项目简介
本项目使用 STM32F103C8T6 单片机，实现了串口通信功能，包括基本收发实验和指令控制外设实验。

## 硬件配置
- 主控芯片：STM32F103C8T6
- 串口：USART1（PA9 TX, PA10 RX）
- 波特率：115200
- LED：PC13
- 蜂鸣器：PB1

## 实验列表

### 实验 6：UART 基本收发实验
功能：接收单个字符并回显

### 实验 7：串口指令控制外设
功能：通过串口发送指令控制 LED 和蜂鸣器
- 发送 '1'：点亮 LED
- 发送 '2'：熄灭 LED
- 发送 '3'：开启蜂鸣器 / - 发送 '3'：开启蜂鸣器响1秒
- 发送 '4'：关闭蜂鸣器

## 遇到的问题与解决方法

### 问题 1：串口没有任何数据输出
**现象**：串口助手看不到任何数据

**原因分析**：
- 使用了外部晶振（HSE）作为时钟源，但开发板可能没有焊接外部晶振
- 外部晶振不起振，程序在 SystemClock_Config() 中失败，进入 Error_Handler() 死循环

**解决方法**：
- 方案一：检查开发板是否有外部晶振，如有则确保正常工作
- 方案二：修改时钟配置，改用内部高速时钟（HSI）
  - 将 PLL 源改为 HSI_DIV2
  - 调整 PLL 倍频系数以获得合适的系统时钟

---

### 问题 2：串口接线接反了
**现象**：发送数据后串口助手看不到任何反应

**原因分析**：
- TX 和 RX 接线接反了
- STM32 的 TX 需要连接到 USB-TTL 的 RX
- STM32 的 RX 需要连接到 USB-TTL 的 TX

**解决方法**：
- 重新接线，确保 TX ↔ RX、RX ↔ TX
- 确保 GND 共地（这一点非常重要！）

**正确接线**：
```
STM32F103C8T6        USB-TTL
    PA9 (TX)  ────>  RXD
    PA10 (RX) <────  TXD
    GND        ────>  GND
```

---

### 问题 3：在中断回调函数中使用 HAL_Delay() 导致程序卡死
**现象**：
- 发送 '3' 后蜂鸣器一直响
- 之后发送 '1'、'2'、'4' 都没有任何反应
- 程序完全失去响应

**原因分析**：
- 在 HAL_UART_RxCpltCallback() 中断回调函数中调用了 HAL_Delay(1000)
- HAL_Delay() 依赖于 SysTick 中断
- 在 UART 中断回调中，SysTick 中断可能不会响应（中断优先级问题）
- 导致程序卡死在延时函数中，无法继续执行

**解决方法**：
- 方法一（最简单）：移除中断回调中的延时，改用手动控制开关（添加指令 '4' 来关闭蜂鸣器）
- 方法二（推荐）：使用标志位 + HAL_GetTick() 在主循环中实现非阻塞延时
  - 在中断中只设置标志位和记录时间
  - 在主循环中检查时间差，执行相应操作

**正确的代码示例**：
```c
// 主循环中
if(beep_flag == 1) {
    if(HAL_GetTick() - beep_start_time >= 1000) {
        HAL_GPIO_WritePin(BEEP_GPIO_Port, BEEP_Pin, GPIO_PIN_SET);
        beep_flag = 0;
    }
}

// 中断回调中
case '3':
    HAL_GPIO_WritePin(BEEP_GPIO_Port, BEEP_Pin, GPIO_PIN_RESET);
    beep_start_time = HAL_GetTick();
    beep_flag = 1;
    break;
```

### 问题 4：使用未初始化数组的 strlen() 导致接收长度错误
**现象**：
- 串口接收行为异常
- 可能收不到数据或接收长度不对

**原因分析**：
- 使用了未初始化的 rx_buffer 数组
- 调用 strlen(rx_buffer) 获取长度时，因为没有字符串结束符 '\0'，返回了随机值
- 导致 HAL_UART_Receive_IT() 的长度参数错误

**解决方法**：
- 使用 sizeof() 替代 strlen() 来获取数组长度
- 或者固定指定接收长度（如 1 个字节）
- 初始化数组为全 0 或空字符串

## 重要知识点总结

1. **中断回调函数原则**：
   - ❌ 不要在中断回调中使用 HAL_Delay()
   - ❌ 不要在中断回调中执行耗时操作
   - ✅ 中断回调应该尽量简短，只做必要的操作
   - ✅ 复杂逻辑和延时处理放在主循环中

2. **HAL_GetTick() 的使用**：
   - 获取系统启动后的毫秒数
   - 可以用来实现非阻塞延时
   - 用法：记录开始时间，在主循环中检查时间差

3. **串口接线注意事项**：
   - TX ↔ RX、RX ↔ TX（交叉连接）
   - 必须共地（GND 连接）
   - 波特率、数据位、停止位、校验位必须一致

4. **时钟配置**：
   - 确认开发板是否有外部晶振
   - HSE 外部晶振通常为 8MHz 或 12MHz
   - HSI 内部时钟为 8MHz，可靠性更高但精度稍低
